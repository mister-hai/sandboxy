Handler for the APISession() class

Provides a wrapper for the API Functions

This is NOT designed with an asynchronous focus

This IS stateful

base challenge is created on server

  • requirements

setauth(config: bool = False, ctfdurl: Optional[str] = None, ctfdtoken: Optional[str] = None, adminusername: Optional[str] = None, adminpassword: Optional[str] = None)¶

Sets variables on the class instance to allow for authentication to the CTFd server instance.

Set to use command line arguments by default, but uses config=True for loading, config does not require URL of CTFd server

Will optionally take a token, or username/password combination BOTH METHODS REQUIRE URL

Parameters

  • config (bool) – If true, uses config file for values if False, uses supplied parameters

  • ctfdurl (str) – URI for CTFd server

  • ctfdtoken (str) – Token provided by admin panel in ctfd

  • adminpassword (str) – admin pass

  • adminusername (str) – admin name

Sync Category:

Maps to the command >>> ctfcli synccategory <categoryname> –ctfdurl <URL> –ctfdtoken <TOKEN> Use after : >>> ctfcli init >>> ctfcli listcategories

Do NOT use –adminusername <NAME> –adminpassword <PASS>

Parameters
